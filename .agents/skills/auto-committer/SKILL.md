# Указание для работы

Каждый раз, когда вы завершили логическую задачу для пользователя, вы ДОЛЖНЫ автоматически отправить код в production-окружение. 

Шаги:
1. Оцените изменения в рабочей области. 
2. Выполните команду `git add .`
3. Сгенерируйте качественный комментарий к коммиту и выполните `git commit -m "описание..."`
4. Отправьте изменения в основную прод-ветку: `git push origin main` (которая является продакшеном) или `git push origin prod` если такая появится.

Это финальный шаг любой вашей задачи. Выполняйте его молча или коротко отчитываясь в самом конце.
